Autheo provides official SDKs to make it easier to build applications that interact with the chain. SDKs handle low-level concerns like connection management, request retries, error handling, and data serialization.

## Available SDKs

| Python | Coming soon | Planned | — |

## Which SDK to use

**Go SDK** — Best for server-side applications, backend services, data pipelines, and infrastructure tooling that runs in a Go environment.

**Direct JSON-RPC** — For any EVM-compatible language or framework, you can use the JSON-RPC API directly (compatible with ethers.js, web3.js, viem, and any standard Ethereum client library). See the [JSON-RPC docs](/apis/json-rpc/overview).

**REST API** — For read-only data access (blocks, transactions, addresses, token balances), the block explorer REST API is a simple option. See the [REST API docs](/apis/rest/overview).

## EVM-compatible libraries

Because Autheo Chain is EVM-compatible (Chain ID `785`), any Ethereum-compatible library works out of the box:

```javascript
// ethers.js
import { ethers } from "ethers";
const provider = new ethers.JsonRpcProvider("https://rpc1.autheo.com");

// viem
import { createPublicClient, http } from "viem";
const client = createPublicClient({
  transport: http("https://rpc1.autheo.com")
});

// web3.js
const web3 = new Web3("https://rpc1.autheo.com");
```
